### 观察者模式
> * 解决问题：当对象间存在一对多关系时，则使用观察者模式（Observer Pattern）。比如，当一个对象被修改时，则会自动通知它的依赖对象。
> * 优点：通过建立一套触发机制可以很方便触发观察者去执行行为。
> * 缺点：通过建立触发机制可以很方便的触发观察对象实施行为
> * **应用场景：**
> * 1. 需要联动 A 变化引起 B变化 ...
> * 2. 观察对象和观察目标之间的关系不明确

### 发布订阅模式
> * 和观察者模式的区别（非要说区别）那就是发布订阅中订阅对象和发布对象之间通过proxy去处理，解耦合，而观察者模式中观察对象和观察目标之间是耦合的
